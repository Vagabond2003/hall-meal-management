# ONLINE HALL MEAL MANAGEMENT SYSTEM
# Comprehensive System Documentation (Exhaustive Detail)

================================================================================
I. CORE ARCHITECTURE & GLOBAL STATE
================================================================================
The application is a Next.js 15 app router web application.
- Authentication: NextAuth.js (v4) with JWT. The session object is customized to include user `id`, `role` (admin/student), `is_approved`, `token_number`, and `meal_selection_enabled`.
- Middleware: Next.js edge middleware protects routes, ensuring:
  - Unauthenticated users go to `/login`.
  - Approved students go to `/student/*`.
  - Admins go to `/admin/*`.
  - Pending/deactivated students go to holding pages.
- Database: Supabase PostgreSQL. Standard schema with Row Level Security.
- Styling: Tailwind CSS, Shadcn UI components, Framer Motion for animations. Extensive use of custom fonts (`font-heading`, `font-sans`).
- Icons: `lucide-react` library.

III. DETAILED SCREEN-BY-SCREEN BREAKDOWN (ADMIN)
================================================================================

1. ADMIN DASHBOARD (`/admin/dashboard`)
---------------------------------------
File: `src/app/admin/dashboard/AdminDashboardClient.tsx`
- Layout & Header: 
  - Title "Admin Dashboard" and subtitle.
  - Action Buttons: "Export [Month] PDF" (Amber outline) and "Export [Month] Excel" (Primary outline). These trigger client-side file downloads using `@react-pdf/renderer` and `xlsx`.
- 4 Quick Stat Cards (Grid):
  1. "Total Students" (Users icon, blue/primary)
  2. "Pending Approvals" (UserCheck icon. Turns Amber if >0, otherwise Primary)
  3. "Active Meals" (Utensils icon, Primary)
  4. "This Month Revenue" (Banknote icon, ৳ prefix)
- Pending Approvals Widget (Width: 2/3 of grid):
  - Header: "Pending Approvals" with an amber badge showing count.
  - List Items: Shows student `name`, `email`, and `token_number`.
  - Actions: Red circle X button (Reject), Green CheckCircle button (Approve). 
  - Empty State: If 0, shows a large Green CheckCircle and "All caught up!".
- Recent Activity Feed (Width: 1/3 of grid):
  - Vertical timeline with a subtle background line.
  - Shows recent events like "New Student" (Blue icon) or "Meal Added" (Amber icon), including the date (e.g. "Mar 15").

2. MANAGE STUDENTS (`/admin/students`)
---------------------------------------
File: `src/app/admin/students/StudentsClient.tsx`
- Header: Title "Student Management".
- Toolbar:
  - Left: Search Bar with debounce. Searches by name, email, or token number.
  - Right: Horizontal pills (All, Pending, Active, Deactivated) mapped to state `filter`.
- Main Data Table:
  - Columns: Token Number, Student Name, Status, Meals (Selection Toggle), Actions.
  - Rows:
    - Token column: Displays `#Token`.
    - Status badge: Pending (Amber), Active (Green), Deactivated (Gray).
    - Meals Toggle Switch: Admin can turn on/off meal selection for an individual student. Grayed out if pending.
    - Actions (Icons): 
      - Approve (Green Check) for pending.
      - Deactivate (Amber ShieldBan) or Reactivate (Green Power) for approved users.
      - Delete Account (Red Trash icon - opens AlertDialog).
      - View Profile (Eye icon - links to detail page).
- Delete Dialog: Standard alert dialog confirming permanent deletion.

2. MEAL SELECTION (`/student/meal-selection`)
---------------------------------------
File: `src/app/student/meal-selection/MealSelectionClient.tsx`
- Welcome Banner: (Reused identical rich gradient banner from Dashboard showing Name and Cutoff).
- Tab Navigation: 
  - Two buttons: "Weekly Menu" (formerly Rolling 7 Days) and "Meal Plan" (formerly Calendar). Default is `week`.
- Weekly Menu Tab View (`activeTab === 'week'`):
  - Vertical list. Iterates through the next 7 days starting from Today.
  - For each day, displays 3 `MealToggleCard` components (Breakfast, Lunch, Dinner).
  - Toggles dynamically evaluate the cutoff time constraints. If passed cutoff, the green switch is replaced with a gray Lock icon meaning the user cannot change their selection.
- Meal Plan Tab View (`activeTab === 'calendar'`):
  - Left Side: Full monthly calendar UI.
    - Dates are grayed out if past cutoff.
    - Dates have green dots underneath the number if meals are selected.
  - Right Side: Details Panel for the clicked date.
    - Shows the specific date header.
    - Renders the 3 `MealToggleCard` components for that specific day.
  - Bulk Action Buttons (Rendered below Details Panel for future dates):
    - Red outline button: "⏸ Turn off all meals from this day onwards".
    - Green outline button: "▶ Turn on all meals from this day onwards".
    - Clicking these triggers `/api/student/bulk-selection` and refreshes the calendar dots immediately.
